# Персональный монитор обновлений сайтов

Label: `wayfinder:map`

## Destination

Сформировать согласованный набор продуктовых и архитектурных решений, достаточный для запуска `/to-spec`, без написания самой спецификации, декомпозиции реализации или разработки приложения.

## Notes

- Рабочий поток: `/wayfinder -> /to-spec -> /to-tickets -> /implement`. По завершении карты явно сообщить пользователю: «Готово к `/to-spec`», но не запускать следующий скилл автоматически.
- Это planning-only effort. Билеты разрешают решения и создают дешёвые прототипы для обсуждения; приложение не реализуется.
- Перед каждой сессией читать [`CONTEXT.md`](../../CONTEXT.md) и применять `/domain-modeling`; для HITL-решений применять `/grilling`, для UI — `/prototype`.
- MVP — однопользовательское локальное веб-приложение на русском языке, официально запускаемое и проверяемое только на Windows вручную ярлыком или командой.
- Около 100 общедоступных страниц; поддерживается JavaScript-рендеринг, ожидание и однократная прокрутка до целевой области. Авторизация, CAPTCHA, антибот-обход, клики и бесконечная прокрутка не поддерживаются.
- Целевой CSS-селектор может вернуть несколько элементов; ноль совпадений — ошибка. Исключения удаляют совпавшие поддеревья целиком. Классы и атрибуты участвуют в поиске, но не сравниваются.
- Снимок содержит нормализованные структуру и видимый текст выбранных элементов. Добавление, удаление, изменение текста и перестановка считаются Изменениями.
- Интервалы: 6, 12, 24, 48 и 72 часа. Есть ручная Проверка, Приостановленный монитор, одна Просроченная проверка после запуска и один повтор через минуту при ошибке. Ручная Проверка сдвигает расписание.
- Первая успешная Проверка создаёт Базовый снимок без уведомления. Изменение URL, целевого селектора или исключений после подтверждения очищает всю историю Монитора; остальные настройки историю сохраняют.
- Каждая Проверка попадает в журнал. Снимки сохраняются только как первая база и при Изменениях; Сравнение показывается в двух колонках в стиле Git-клиента.
- Обязательны уведомления внутри интерфейса, через открытую/фоновую вкладку браузера и через существующий Telegram-модуль. Глобальный Режим контрольных уведомлений добавляет Telegram- и браузерные сообщения для успешных Проверок без Изменений.
- У Монитора может быть несколько Меток; группы отсутствуют. Список фильтруется по выбранной Метке.

## Decisions so far

<!-- Один контекстный указатель на каждый закрытый билет; подробное решение живёт только в билете. -->

- [Выбрать архитектуру и технологический стек](issues/01-choose-architecture-and-stack.md) — выбран однопроцессный TypeScript-монолит на Node.js/Fastify с React/Vite, Playwright Chromium и SQLite; границы модулей сохраняют последующие решения независимыми.
- [Определить модель загрузки и извлечения страницы](issues/02-define-page-loading-and-extraction.md) — выбран ограниченный по времени Playwright-конвейер с ожиданием устойчивой Целевой области, одной прокруткой и атомарным извлечением строгих CSS-совпадений после Исключений.
- [Определить формат Снимка и алгоритм Сравнения](issues/03-define-snapshot-and-comparison.md) — выбран версионированный канонический JSON структуры и rendered-текста; равенство определяется байтами, а ограниченный Myers diff строит двухколоночное Сравнение и показывает перестановки как удаление с добавлением.
- [Спроектировать выполнение и планирование Проверок](issues/04-design-check-execution-and-scheduling.md) — выбрана долговечная SQLite-очередь с одним worker, сроками от завершения цепочки, ровно одной Повторной проверкой, объединением дублей и восстановлением после простоя или сбоя.
- [Спроектировать локальное хранение и жизненный цикл истории](issues/05-design-local-persistence.md) — выбрана единая SQLite-модель, где каждая Проверка остаётся в журнале, Снимки сохраняются только для базы и Изменений, Сравнения вычисляются по ссылкам, а смена Области наблюдения атомарно очищает прежнюю Историю Монитора.
- [Получить и проверить контракт Telegram-модуля](issues/07-inspect-telegram-module.md) — подтверждён одноразовый Python sender `telegram-alert-bus`: child process принимает строгий JSON через UTF-8 stdin, использует Windows keyring и exit codes, но не предоставляет очередь, дедупликацию, атомарный multipart или общий deadline.

## Not yet specified

- Совместимость и упаковка зависимостей на Windows, которые зависят от выбранного технологического стека.
- Уточнения пользовательских потоков, которые проявятся только после реакции на дешёвый UI-прототип.

## Out of scope

- Написание спецификации, создание реализационных билетов и реализация приложения в рамках этой карты.
- Облачный или постоянно доступный серверный режим; macOS и Linux; публикация на GitHub; автоматический запуск при входе в Windows.
- Авторизованные страницы, CAPTCHA, обход антибот-защиты и хранение учётных данных.
- Клики по странице, кнопочная пагинация, бесконечная прокрутка и сценарии взаимодействия.
- Интерактивный выбор DOM-элемента кликом; в MVP селекторы копируются через DevTools и проверяются предпросмотром.
- Целевые области внутри iframe или shadow DOM; CSS-селекторы MVP применяются только к light DOM главного документа.
- Web Push и браузерные уведомления после закрытия вкладки.
- Группы Мониторов и международный интерфейс.
