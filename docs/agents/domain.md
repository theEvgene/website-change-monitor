# Domain Docs

Проект использует один предметный контекст.

## Before exploring, read these

- `CONTEXT.md` в корне проекта.
- Релевантные ADR в `docs/adr/`, если директория существует.

Если файла или директории пока нет, продолжить без предупреждения. Они создаются лениво при появлении новых терминов или архитектурных решений.

## File structure

```text
/
├── CONTEXT.md
├── docs/
│   └── adr/
└── src/
```

## Use the glossary's vocabulary

В задачах, предложениях, гипотезах и тестах использовать термины из `CONTEXT.md` и избегать перечисленных там синонимов. Реальный пробел в языке передать в `/domain-modeling`.

## Flag ADR conflicts

Если предлагаемое решение противоречит существующему ADR, явно указать конфликт, а не переопределять решение молча.
